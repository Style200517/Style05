# SignalFlow Copywriting Rules

## 0. Purpose

SignalFlow는 투자 추천 서비스가 아니라 주식 신호 리서치 도구다.

따라서 UI 문구는:
- 데이터 기반
- 설명 가능
- 리스크 동반
- 관찰 중심
- 과장 없는 표현
이어야 한다.

---

## 1. Core Positioning

Use this positioning:

- 주식 신호 분석 도구
- 관찰 후보 탐색
- 상승 가능성 점수
- 뉴스/공시/가격/거래량 기반 분석
- 과거 데이터 기반 백테스트
- 리스크 플래그 제공

Do not position as:
- 매수 추천 서비스
- 수익 보장 서비스
- 급등주 추천기
- 자동매매 서비스
- 투자자문 서비스

---

## 2. Forbidden Live Signal Copy

Do not use these in live signal areas:

- 매수
- 매도
- 강력 매수
- 추천 매수
- 오늘 살 종목
- 급등주
- 급등 확정
- 수익 보장
- 무조건 상승
- AI 추천주
- 대박 종목
- 고수익 보장
- 손실 없음
- 확정 수익
- 반드시 상승
- 지금 사야 함
- 목표 수익 확정

If reference image contains "매수", preserve the visual chip style but replace text with:
- 강한 관찰
- 관찰
- 주의

---

## 3. Approved Live Signal Labels

Use these labels:

### Primary signal labels

- 강한 관찰
- 관찰
- 중립
- 주의
- 제외

### English internal enum mapping

- strong_watch = 강한 관찰
- watch = 관찰
- neutral = 중립
- risk = 주의
- exclude = 제외

### Usage

강한 관찰:
- 점수와 신뢰도가 높고 리스크가 제한적인 후보
- 절대 "매수"의 의미로 설명하지 않는다.

관찰:
- 추가 모니터링 가치가 있는 후보

중립:
- 신호가 뚜렷하지 않음

주의:
- 변동성, 악재, 과열, 유동성 등 리스크가 큼

제외:
- 현재 조건상 후보에서 제외

---

## 4. Score Copy

Use:
- 상승 가능성 점수
- 신뢰도
- 위험도
- 기대수익률
- 하락 위험
- 점수 구성
- 리스크 플래그
- 데이터 품질
- 과거 유사 패턴
- 백테스트 기준

Do not use:
- 상승 확정
- 수익 확률 보장
- 성공 보장
- 반드시 수익
- 무조건 오른다

Preferred wording:
- "상승 가능성 점수"
- "통계적 후보"
- "과거 유사 패턴 기준"
- "데이터 기준 관찰 후보"
- "리스크 확인 필요"

---

## 5. Risk Copy

Always display risk with signals.

Approved risk phrases:
- 단기 급등 후 추격 위험
- 거래대금 부족
- 변동성 확대
- 악재성 뉴스 발생
- 공시 불확실성
- 실적 발표 전후 변동성
- 시장 전체 하락 압력
- 이미 주가에 반영됐을 가능성
- 데이터 부족으로 신뢰도 제한
- 유동성 리스크

Do not hide risk because a stock has high score.

- 공시
- 수급
- 실적
- 매크로

- 관찰

News status labels:
- 주목
- 관찰
- 경계
- 확인
- 중립

Do not:
- "매수 호재"
- "급등 호재"
- "확정 호재"
- "대박 뉴스"

## 7. Watchlist Copy

Page title:
- 관심종목 관리

Subtitle:
- 관심 종목의 신호, 변동률, 알림 상태를 한눈에 관리

Approved section names:
- 내 관심종목 목록
- 관심종목 요약
- 알림 센터
- 내 그룹
- 관심종목 성과 추이
- 신호 변화 히스토리
- 우선 확인 종목

Alert status:
- 활성
- 가격근접
- 이슈발생
- OFF

Do not use:
- 매수 알림
- 매도 알림
- 급등 알림 확정

Use instead:
- 신호 알림
- 가격 알림
- 이슈 알림
- 리스크 알림

---

## 8. Backtest Copy

Backtest is historical analysis.

Approved:
- 백테스트 분석
- 전략 성과와 리스크를 과거 데이터 기준으로 검증
- 누적 성과
- 백테스트 요약
- 누적 수익률
- 벤치마크 대비
- 승률
- 최대 낙폭
- 샤프지수
- 거래 횟수
- 기간별 수익률
- 리스크 분석
- 연도별 성과 비교
- 최근 체결 예시

Historical trade result labels allowed:
- 익절
- 손절
- 보유
- 청산
- 완료
- 실행중

Mandatory disclaimer:
- 과거 성과는 미래 수익을 보장하지 않으며, 수수료·슬리피지 반영 기준입니다.

Do not:
- "이 전략은 수익이 보장됩니다"
- "실전에서 그대로 수익"
- "무조건 이기는 전략"

## 11. Number Formatting

Korean stocks:
- Price: `81,600`
- Trading value: `23,610억`
- Market cap: `486조`
- Percent: `+1.24%`, `-0.21%`

US stocks:
- Price: `124.56`
- Trading value: `28.4B`
- Market cap: `3.05T`
- Percent: `+2.45%`, `-1.25%`

KPI:
- Use larger type for important values.
- Positive values use red/orange.
- Negative market movement uses blue.
- Risk drawdown uses blue.

---

## 12. Empty State Copy

Use:
- 표시할 데이터가 없습니다.
- 조건에 맞는 결과가 없습니다.
- 아직 수집된 뉴스가 없습니다.
- 백테스트 결과가 없습니다.
- 관심종목을 추가해 주세요.

Avoid:
- Nothing here
- No data
- Failed
- Error only

---

## 13. Error State Copy

Use:
- 데이터를 불러오지 못했습니다.
- 잠시 후 다시 시도해 주세요.
- 외부 데이터 제공 지연으로 일부 정보가 표시되지 않을 수 있습니다.
- 다시 시도

Avoid:
- raw API error
- stack trace
- undefined
- null

---

## 14. Loading Copy

Use minimal loading text only where needed:
- 불러오는 중
- 데이터 업데이트 중

Prefer skeleton UI over text-only loading.

---

## 15. Disclaimer Copy

Short disclaimer:
- 본 화면은 투자 판단 보조용 리서치 정보이며, 투자 손실 가능성이 있습니다.

Backtest disclaimer:
- 과거 성과는 미래 수익을 보장하지 않으며, 수수료·슬리피지 반영 기준입니다.

News disclaimer:
- 뉴스 영향도는 기사 내용, 출처, 시장 반응을 종합한 참고 지표입니다.

Score disclaimer:
- 상승 가능성 점수는 과거 데이터와 현재 신호를 기반으로 한 참고 지표입니다.

Use disclaimers subtly. Do not overload every card, but include where appropriate.

---

## 16. Final Copy Rule

Every page must balance:
- signal
- reason
- risk
- confidence

Do not show positive signal alone.
Do not imply certainty.
Do not hide uncertainty.
